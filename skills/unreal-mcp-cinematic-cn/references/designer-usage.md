# 设计师使用说明

## 刚安装完怎么问

第一次可以直接说：

```text
我刚安装 ue5-cinematic-shot-language-cn，请告诉我怎么把设计师给的分镜、镜头表或分段画面描述交给 AI 执行到 UE5 Sequencer。
```

如果已经有分段画面描述，直接说：

```text
用快速动态模式，按这份分段画面描述执行。复用当前场景，不重搭。
```

## 设计师最好提供什么

- 分镜或分段画面描述：每段发生什么，最好有时间段。
- 当前场景截图：能看清主体、空间关系和想拍方向。
- 交付级镜头最好先让 AI 做场景勘察：只读识别主要可见 Actor 的名称、位置、大小、可见性和用途候选，再确认核心目标。
- 目标类型：场景展示、角色登场、技能演出、Boss 登场、买量视频、剧情过场或实机演示。
- 时长：例如 10 秒、20 秒、30 秒。
- 画幅：最好说明 16:9、9:16、1:1、4:5 或项目比例；AI 会先读现有项目设置，交付比例仍不明确时再问。
- 角色朝向：最好说明本地正脸轴，例如 `+X`、`-X`、`+Y` 或 `-Y`。未说明时 AI 可以只读检查组件/骨架关系和候选方向截图，但看不清或标准制作风险较高时仍会询问。
- 不许改的内容：剧情、爽点、角色关系、场景摆放、已有 Actor。
- 允许 AI 自主补齐的技术内容：未指定的机位坐标、构图微调、焦距、焦点、Camera Path、曲线、避障、必要修正 Key 和验片检查；占位资产或动作仍需设计师授权。这些补齐不得改变镜头数量、剧情、角色行为、爽点和最终记忆画面。
- 最终记忆画面：最后想让观众记住什么，或哪一帧适合截图。
- 录屏请使用全屏或标准分辨率，并尽量锁定帧率；窗口局部录屏可能导致验片误判。

## AI 应该做什么

- 理解设计师给的镜头，而不是重写创意。
- 把文字分镜转成 Sequencer 执行规格。
- 补齐技术执行字段：CineCameraActor、CameraCut、焦距、Look At、关键帧、绑定 Actor。
- 如果资源缺失，先报告缺口；只有我授权占位时才使用占位资产或动作。
- 执行后检查遮挡、穿模、浮空、过曝、黑屏、UI 外露和主体是否可读。

## 五种模式

- 静态分镜：确认单/多镜头构图和 CameraCut，不制作运动。
- 快速单镜头：只处理一个镜头、一个 Actor、一个运镜原语或一个技术点；只写发生变化的维度，复用未变化的 CameraCut、焦距和焦点，默认只做一次结构读回，画面由设计师手动看。
- 快速动态：为一个或多个已给镜头制作动态粗版，并做低成本关键画面快筛。
- 标准制作：完成设计师要求范围内的路径、控制权、曲线、焦距/焦点、状态和风险帧修正；没有播放证据时仍不能确认流畅。
- 成片检查：必须基于真实播放、录屏或渲染证据；没有证据时只能做成片预检，不能判定通过。

五种模式的完整验收矩阵和允许完成措辞见 `mcp-cinematic-execution.md`，本文件不维护第二套完成标准。

## 推荐流程

1. 设计师先给分镜、脚本、表格或截图。
2. 标准制作或成片检查前，如果核心目标不明确，先让 AI 通过 MCP 做只读场景勘察，列出 5-10 个候选后再确认主角。
3. 可以直接描述目标，不必记住模式名称；AI 会按意图选择最低可满足的制作深度并报告。
4. 单镜头/单技术点通常使用快速单镜头，多镜头粗版通常使用快速动态，“正式/标准/制作级”进入标准制作。
5. 准备录屏或交付时，基于连续播放、录屏或渲染进入成片检查。
6. 返工时只指出执行问题：节奏太平、主体太小、镜头太高、遮挡、穿模、方向感不清楚等。

## 短提示词

单镜头想快速制作，可以直接说：

```text
用快速单镜头模式，只制作这个镜头。不截图、不导出预览、不全片播放，只写入关键帧并做一次结构读回；画面我手动播放判断。
```

```text
用快速动态模式，按这份分镜做。复用当前场景，不重搭。
```

```text
先通过 MCP 做只读场景勘察：按可见性、位置、大小和空间意义列出 5-10 个可拍 Actor 候选，分类为核心主体、前景层次、路径参考、潜在遮挡物和背景参照。不要全量暴力遍历，不移动或生成场景 Actor。等我确认核心目标后，再进入标准制作。
```
需要更明确时再补一句：

```text
只补 Sequencer 执行细节和验片点，不改我的剧情、爽点和镜头意图。
```

## 有分段画面描述时

```text
使用 ue5-cinematic-shot-language-cn，根据这份分段画面描述执行。每个时间段都按原文意图保留。复用当前场景，用快速动态模式先粗做一版。允许 AI 把我给出的镜头运动转成机位、路径、关键帧、CameraCut 和必要占位规格，但不要新增镜头，不要重搭场景，不要移动无关资产，不要改我的故事和爽点。先输出 Sequencer 执行规格，再通过 VibeUE / UE MCP 执行。
```

如果描述里的角色、特效或道具资产暂时不存在，补一句：

```text
如果描述里的特效或角色资产场景里不存在，先报告缺口；只有我授权占位时才使用可读占位表达节奏。
```

## 按需求补一句

- 城市/地图展示：按我给的镜头展示规模、路径、地标、区域层次和最终收束画面。
- 角色登场：按我给的身份揭示、局部细节、标志动作和最后姿态执行。
- 技能演出：按我给的蓄力、释放、命中反馈和余波执行。
- Boss 登场：按我给的异常预兆、尺度压迫、威胁动作和玩家目标执行。
- 买量视频：按我给的开头钩子、主要爽点、故事冲突、画幅比例和最终记忆画面执行；项目和上下文都无法确定交付比例时，AI 必须先问。
- 游戏视角：按我指定的第一人称、第三人称、MOBA 俯视、固定镜头或过场镜头执行。


## 常用镜头语言测试

如果设计师说“测试镜头语言能力”或“给我一组常用运镜测试提示词”，AI 可以生成一组短测试分镜。时长按动作、信息量、镜头职责和总时长分配，不默认等长；用快速动态模式，复用当前场景。
设计师也可以直接这样问：

```text
帮我生成一组常用镜头语言测试提示词，用来测试这个 Skill 的运镜能力。
```

建议先测这 5 个：

- 推镜 Push In：测试主体推进、缓入缓出和主体可读性。
- 拉镜 Pull Out：测试从主体到环境关系的揭示。
- 半环绕 Orbit：测试 Camera Path / Look At Path 分离和空间关系变化。
- 希区柯克变焦 Dolly Zoom：测试摄像机位置和焦距联动，主体大小相对稳定，背景空间变化明显。
- 低机位掠过揭示 Low Reveal：测试前景短遮挡、低机位穿行和最终构图落稳。

生成测试提示词时必须写明：普通运动优先少量主控 Key；只有避障、重锁构图、前景掠过或明确节奏点时才加修正 Key。完成后报告卡顿、漂移、遮挡、穿模、纯天空/纯地面和主体不可读问题。

## 返工怎么说

- 节奏问题：按原分镜不变，只调整 CameraCut 时长、停顿和速度变化。
- 镜头问题：不改剧情，重做机位、焦距、Look At、关键帧和落稳。
- 骨架太简单：结构创建成功，但镜头仍过于简单。保持原分镜，按标准制作补齐路径、节奏、曲线、构图变化、焦点/Look At 和结尾落稳；不要只加中间 Key。
- 构图问题：主体不清楚，调整景别、焦距、前景遮挡和主体占比。
- 游戏问题：方向感不清楚，结尾镜头要交代玩家下一步看哪里。
- 买量问题：如果钩子或爽点不对，设计师先改创意方向，AI 再重做镜头表达。


## AI 什么时候主动问你

AI 会优先按任务措辞选择最低可满足的模式，不要求设计师记住模式名称。“测试/试做/先跑通”通常进入快速动态；“正式/标准/制作级”进入标准制作；“直接做/不用问”且目标清楚时直接执行并报告推断。

只有模式差异会显著改变写入范围、耗时或验证成本，且上下文仍无法推断时，AI 才询问：

```text
你是想用静态分镜确认构图，快速单镜头处理一个镜头/技术点，快速动态制作动态粗版，标准制作完成完整写入与关键画面快筛，还是基于真实播放/录屏/渲染做成片检查？
```

这一步只确认制作深度，不改变创意；已经明确模式、可从措辞推断，或延续当前已确认模式时，不重复询问。

## 什么时候让 AI 发挥

只有在设计师明确说“你帮我补一版分镜”“你来发挥”“我没有镜头方案”时，AI 才能主动提镜头创意。默认情况下，AI 执行设计师给的镜头。
