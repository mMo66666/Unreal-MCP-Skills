# CG 预览与导出验收

区分结构成功、画面成功、运动成功和成片级成功。不要把编辑器操作画面当成最终 CG。

## 四层验证

- 结构验证：LevelSequence、帧率、Playback Range、CameraCut、摄像机绑定、关键轨道 Section Range 正确。
- 画面验证：关键帧没有黑屏、白屏、纯天空、纯地面、纯墙面、穿模、严重遮挡、主体不可读。
- 运动级验证：使用连续播放、录屏、渲染或具备时间连续性的等价分析检查卡顿、顿挫、速度连续性、结尾漂移和切点节奏。
- 成片级验证：无编辑器 UI、无 Sequencer 面板、无工具栏/状态栏、无警告文字、无选择框、无 Gizmo、无调试线。

## 预览验片链

先检查当前工具是否支持连续播放、视频读取、时序分析或渲染结果访问；支持且操作安全时主动使用并提升验证层级，不支持时才退回结构读回、风险抽帧和设计师验片。快速单镜头默认只做第 1 步结构读回，除非用户要求或问题需要更深验证。

当 MRQ 可用且需要定位运动异常时，可按 `mrq-temporal-diagnostics.md` 渲染连续 PNG 风险窗口；只有帧号连续、覆盖相关运动并完成时序比较时，才能作为对应区间的运动级证据。

1. 结构读回：先验证时长、CameraCut、绑定、Section Range 和资产保存。
2. 低清风险帧截图：优先检查切镜两侧、运动峰值、最近障碍、焦点变化、动作关键姿势和结尾稳定区；没有特定风险时再用起始/中段/结束前帧作默认抽查。
3. 运动验片：有连续影像且 AI 具备时序分析能力时，AI 可做技术检查并说明方法；否则由设计师正常速度播放后反馈卡顿、顿挫、漂移和节奏。
4. 成片级检查：基于正式录屏或渲染检查 UI、黑白屏、遮挡、穿模和主体可读性；主观节奏仍需正常速度观看证据。

如果第 3 步没有可用的连续影像分析或设计师播放反馈，不要卡死流程；报告当前只能确认结构和关键帧画面，不能确认运动流畅性。

## 预览规则

- 结构验证通过不等于镜头成立。
- 画面截图通过不等于成片级预览通过，也不等于运动流畅。
- 带 UI 的编辑器录屏只能辅助判断构图和节奏，不能作为正式验收。
- 正式预览优先使用 CameraCut、Game View、无 UI 截图或正式渲染。
- 只有要把结论升级为“运动通过”时，才必须有连续播放、录屏、渲染分析或设计师反馈。快速单镜头与快速动态可以按验收矩阵停止在结构读回或关键画面快筛，但必须明确运动尚未验证；只有抽帧和数据时不能宣称“流畅”。

## 抽帧复核

- 验片优先使用正式录屏或渲染结果逐帧/按时间复核，并说明素材来源和分析方法。
- 没有视频文件时，用真实播放并按时间截图。
- 浏览器或播放器 seek 截图只能辅助；如果多张截图完全一致，先复核 currentTime 是否真实变化，不要立刻判定视频静止。
- 动画或 Sequencer 大幅跳帧后立即截图，可能残留 TAA、TSR 或 Motion Blur 的历史重影。先等待画面稳定或临时关闭相关视口效果复核，并恢复原设置；不得把这类时域伪影直接判成动画、相机或成片重影。
- 抽帧异常时，用真实播放截图或其他工具复核后再下结论。

## 观众视角验片

- 这一秒观众应该看哪里，主体是否明确。
- 本镜头是否提供了新信息，还是只重复上一镜。
- 节奏是否有呼吸：该快的快，该停的停，重要画面是否留够阅读时间。
- 标准制作或成片检查中，如果观众只能看到墙面、天空、地面、强光或杂乱前景，且不符合设计师要求的主体或信息目标，判定需要返工。快速单镜头或快速动态的技术测试只记录画面语义局限；除非它妨碍当前技术目标验证，否则不自动返工或升级验证。

## 验收措辞

- 结构通过：仅完成结构读回，画面与运动尚未验证；常用于快速单镜头。
- 关键画面快筛通过：只能确认抽查帧的构图、遮挡、穿模、曝光和主体可读性，不能确认运动。
- 运动通过：必须有连续播放、录屏、渲染或等价时序证据，并完成速度、卡顿、漂移和切点检查；仅抽帧不满足。
- 成片检查通过：结构、画面、运动和成片级检查全部通过；这是唯一允许使用不带限定“通过”的状态。
- 部分通过：必须列出具体通过层级和缺失证据，不能只写笼统的“基本通过”。
- 失败：UI 遮挡主体、画面为空、黑屏、白屏、穿模、长时间遮挡、主体不可读，或运动预览出现明显卡顿/漂移。

模式对应的允许完成措辞以 `mcp-cinematic-execution.md` 的执行模式验收矩阵为准。
