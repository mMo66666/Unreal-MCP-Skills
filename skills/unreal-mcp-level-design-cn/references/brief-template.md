# 地编简报模板

仅在任务需要规划或实施时读取。先从用户描述和当前关卡自动填充；不要把整张表原样丢给设计师。

## 必填判断

| 项目 | 要回答的问题 | 缺失时的安全默认 |
|---|---|---|
| 目标地图 | 修改哪个 World/关卡？ | 使用当前打开关卡；正式资产不明确时只读或写测试副本 |
| 任务类型 | 规划、评审、诊断、搭建还是改造？ | 动词含“做/搭/改/生成”视为搭建，其余先只读 |
| 体验类型 | 步行探索、FPS、载具、竞速、展示还是叙事？ | 沿用现有关卡可推断的玩家尺度，不强行套 FPS |
| 场景原型 | 镇中心、住宅区、工业区、荒野等原型及主要功能分区？ | 从现有资产和目标推断；无法判断时先做可逆灰盒 |
| 修改范围 | 允许动地形、道路、建筑、植被、灯光到什么程度？ | 只做请求直接涉及的系统 |
| 交付深度 | 灰盒、可玩原型、可信场景还是展示级？ | 先交付可读灰盒与验证证据，不越级宣称完成 |

## 通用尺度参考

只有项目没有自己的 metrics 时才采用；已有角色胶囊、移动速度、套件网格或设计规范优先。

- UE 世界单位按厘米理解，`100 uu = 1 m`。
- 从玩家胶囊、门、楼梯、车辆和已有模块反推尺度，不凭空认定人物尺寸。
- 步行体验用实际移动速度换算行进时间：`时间 = 路径长度 / 平均速度`。
- 先验证“能通过、看得懂、走得通”，再增加装饰密度。
- 门洞、走廊、楼梯、掩体等指标必须相对当前玩家胶囊与镜头验证，不硬编码单一游戏标准。

## 默认命名

- 测试根目录：`/Game/MCP_Test/LevelDesign/<任务名>`
- Actor 前缀：`LD_`
- 临时 Data Layer：`DL_LD_<任务名>`
- 阶段标签：`MCP_LD_S01`、`MCP_LD_S02`……

沿用项目已有命名规范时，不创建第二套标准。
