# 镜头组接

本文件用于检查和翻译设计师已经给出的镜头关系。未获“补分镜 / 你来发挥”授权时，只指出连续性、切点或信息重复问题，不主动插镜头、删镜头、重排顺序或改剧情节拍。

多镜头段落只检查设计师已给镜头之间的信息、动作、方向和切点关系，不套“建立到动作”“细节到揭示”或“平静到爆发”等固定顺序。

## 镜头连续性自查

设计多镜头段落时，每个镜头都要能说清它和前后镜头的关系：

- 上一镜把观众的视线、动作、方向或情绪带到哪里。
- 本镜新增什么信息，不能只重复上一镜的画面。
- 下一镜为什么在这里切，是因视线、动作、结果、反应还是情绪变化。
- 如果说不清连接理由，向设计师指出连接问题，并建议调整镜头顺序、景别或切点；不要默认替设计师改顺序。

## 连续性规则

### 动作匹配

- 在动作过程中切镜，比动作完全停止后切更顺。
- 切镜前后动作方向要能接上。
- 快动作可以少量省略，但不能让观众丢失因果。

### 视线匹配

- 角色看向某处后，下一个镜头通常展示被看的对象。
- 如果不展示，会制造悬念或不安。
- 视线方向要和空间关系一致。

### 方向匹配

- 主体向画面右移动，下个镜头也尽量保持向右，除非明确改变方向。
- 追逐、发射、飞行、移动镜头尤其要保持方向。
- 跳轴会让动作变乱，除非是有意制造混乱。

### 景别递进

- 建立信息时由大到小：远景/全景到中景/近景。
- 情绪增强时由中到近：中景到近景/特写。
- 结果揭示时可以由近到大：特写到全景/远景。
- 相同景别是否重复，只在它妨碍设计师给出的信息递进时标记；不为制造变化而强制换景别。

## 切点规则

- 当前镜头主要信息已经被观众读懂，再切。
- 情绪变化处可以切。
- 视线落点处可以切。
- 动作发力或落点处可以切。
- 音乐、音效、粒子爆点可以作为切点，但不能替代叙事理由。

## 时长分配

设计师未给逐镜时长时，按动作、信息量、总时长和切点关系选择均分或非均分，并报告重要假设；不把任一种分配方式设为默认正确答案。

## 常用镜头功能

- 建立镜头：交代空间和关系。
- 动作镜头：展示动作本身。
- 插入镜头：强调手、道具、机关、按钮、武器、表情细节。
- 反应镜头：展示动作意义。
- 过肩镜头：建立对峙、对话、观察关系。
- 空镜/环境镜头：转场、情绪停顿、余韵。
- 余波镜头：展示影响范围和动作重量。

## 输出要求

复杂组接、返工诊断或交接需要时，说明镜头之间的连接依据：

- 因视线切
- 因动作切
- 因情绪切
- 因空间揭示切
- 因结果/反应切
