# CG 调度、灯光与画面安全

场景层次、主体调度、灯光曝光和景深只在当前镜头目标需要时参与判断。

## 三层画面

- 前景层：树、墙边、门框、柱子、车辆、特效边缘。可用于深度、速度、限制信息或设计师要求的遮挡。
- 中景层：角色、Boss、道路、核心建筑、技能命中点。通常承载主要信息。
- 背景层：山体、天际线、远处建筑、天空轮廓。用于尺度、方向和氛围。

## 调度规则

- 主体入画、出画、转身、蓄力、爆发、反应都要对应镜头节拍。
- 前景遮挡是否过长按镜头时长、主体阅读和设计目的判断；只有妨碍当前信息目标时才判失败。无明确设计依据时，主体连续约 0.5-1 秒或短镜头约 25% 时长不可读可作为风险警报，不把经验数字当绝对失败线。
- 没有真实 Actor 时，只有设计师明确授权占位，才可用基础几何体表达主体、路径、阻碍、目标和背景尺度，并标明替代关系；未授权时只报告资产缺口。
- 不要为了镜头好看移动用户已有场景资产；默认只调摄像机和 Sequencer。

## 角色构图高度

- 上三分之一、中心、低位或大留白都是构图候选；按设计师景别、角色姿态和画面目的选择，不用固定眼线或头顶留白比例判定失败。
- 设计师要求平视、中景或动作完整可读时，检查是否因无意高俯视造成主体压低、肢体裁切或动作难读。
- 修正偏高构图时，优先下调摄像机高度、Look At 目标或构图中心，再微调焦距和距离；不要用堆 Rotation Key 解决。

## 灯光曝光

- 强光、发光主体、爆炸、技能特效不能长时间怼镜头造成白屏。
- 夜景需要阅读主体或路径时保留必要轮廓和方向；设计师要求剪影、不可见或信息限制时按原意执行。
- 如果使用闪白转场，必须短，并标注为有意闪白。
- 结尾应完成设计师给出的主体、环境、余韵或抽象画面目标；不强制每个结尾都保留人物或物体主体。

## 景深和焦点

- 景深服务主体，不要把地图路径、动作结果或角色表情糊掉。
- 快速制作可少用复杂景深，先保证构图和运动成立。
- 成片检查时再细查焦点、景深、曝光和轮廓是否破坏信息读取。
