# Skill 回归测试

修改本 Skill 后运行与受影响规则相关的案例；跨领域修改覆盖各受影响类别的代表案例，不机械要求每次运行全部。环境允许且不会修改生产系统时，可让独立 Agent 做无答案泄漏的前向测试；独立 Agent 不是完成修改的硬门槛。

发布前最小哨兵集为 T02、T06、T08、T16、T24、T37、T43；涉及 CameraCut Blend 时追加 T36，涉及 MRQ 时追加 T28 与 T45。哨兵集只防核心能力回退，不替代本次受影响规则的对应案例。

## 通过标准

- 回答必须执行设计师已给意图，不补未授权创意。
- 技术规格必须可写入、可读回、可解释失败原因。
- 没有真实运动或成片证据时，不得升级验证状态。
- 任一案例出现“结构存在即可完成”、多算一帧、双重控制或可见回位跳变，整次修改判为失败。

## T01 创意边界

输入：`20 秒单镜头，主体始终正中，无前景，正面匀速直推，不停顿，不落稳，不授权补分镜。`

通过：保持正中、无前景、匀速直推；不改弧线、不补阶段、不因“电影感”改变路径。

## T02 帧区间

输入：`总长 104 帧，三段 0-28、28-72、72-104。`

通过：正规化为 Playback `[0,104)`；Cuts `[0,28)`、`[28,72)`、`[72,104)`；持续帧数 28/44/32；最后有效帧 103；无空洞或重叠。

## T03 动画采样点

输入：`总长 507 帧，不改 Play Rate；动画接口返回 508 个帧/采样点。`

通过：保持 `[0,507)` 和 Play Rate；把 508 单独报告为源采样/姿势计数，先核对时长和采样率，不静默扩成 508 帧或裁动画。

## T04 状态恢复

输入：`角色在镜头中移动，结束后恢复进入序列前位置；最后画面必须保持最终姿态，不能出现回跳。`

通过：使用 Restore State/Pre-Animated State；最后有效帧保持最终姿态，不 K 回原 Transform；区分画面内最后状态和序列停止后的恢复。

## T05 焦点生效

输入：`近景推镜，ManualFocusDistance 轨道存在，但 Focus Method 仍为 None/Tracking。`

通过：判定焦点未生效；设置并读回 Manual Focus Method、语义焦点目标、厘米距离、光圈和风险帧清晰度，不能只报轨道存在。

## T06 模式边界

输入：`一个镜头，要求标准制作；没有播放或录屏反馈。`

通过：不能降级为快速单镜头；完成完整标准写入和关键画面快筛，措辞只能是“标准制作写入完成……运动流畅性待人工验片”。

## T07 标准制作骨架

输入：`LevelSequence、CameraCut、CineCamera 和起止 Key 已创建，未处理焦点、控制权、曲线、状态和风险帧。`

通过：判为 Sequencer 骨架或未完成标准制作；不得靠增加无目的中间 Key 伪装完成度。

## T08 相机控制权

输入：`Rail 控制路径，同时 K 世界 Location；动态 Look At 开启，同时有密集 Rotation；Manual 与 Tracking Focus 同时存在。`

通过：判定三类双重控制；逐项选择唯一位置、朝向和焦点主控，写明切换帧、关闭旧控制并读回。

## T09 CameraCut 与 Binding

输入：`三段 CameraCut Section 都存在，但首段晚于 Playback Start，一段 Binding 只剩 Guid，切点未抽查。`

通过：判为结构失败；检查头尾覆盖、相邻边界、`EndFrame-1/StartFrame` 两侧画面、Binding 解析和保存重载。

## T10 原生回退

输入：`MCP 创建 CameraCut 时 Guid 转换失败，但 LevelSequence、Camera Binding 和半成品 Section 仍存在。`

通过：先读回并复用半成品；按原生 API 回退，不重复创建；保存重载后验证。只有状态未知、原生 API 不可用或有正式资产风险时停止。

## T11 过度承诺

输入：`结构和关键截图都正常，没有真实播放、录屏或渲染。请确认运镜和成片已经完全合格。`

通过：拒绝确认运动流畅或成片通过；分别报告结构、关键画面快筛和缺失的运动/成片证据。

## T12 技术自主补齐

输入：`20 秒、6 个 CameraCut 的小镇紧急事件分镜已经给定，但没有提供每台相机的精确坐标、焦距、焦点距离、避障点和曲线。按标准制作直接执行，不要改剧情和镜头数量。`

通过：不要求设计师逐项补技术参数；在只读勘察后自主确定机位、焦距、焦点、Camera Path、曲线和有职责的修正 Key，并报告重要假设。不得新增剧情、镜头、角色行为或改变给定切点。

## T13 模式自动选择

输入：`按这份分镜做正式制作级 Sequencer，直接做。`

通过：自动选择标准制作并执行，不因用户没有说出“五种模式”之一而停下询问；结果报告采用标准制作的验证边界。

补充输入：`做一段有动感的小女孩舞蹈 MV，直接做，其他按你的审美发挥。`

通过：把“发挥”“直接做”视为创意授权，自主补齐分镜和技术规格；只有仍缺少会改变角色方向、资产安全或交付规格的关键信息时才询问。

## T14 不确定信息门槛

输入：`角色近景需要正脸，但用户没给正脸轴；Actor Forward 为 +X，MeshComponent 有额外旋转，候选截图被头发遮住。`

通过：不得仅凭 Forward Vector 猜测；先报告只读证据仍不充分，再询问设计师确认 `+X/-X/+Y/-Y`。如果用户已明确正脸轴，则不得重复询问。

## T15 连续影像边界

输入：`AI 只有结构数据和起始/中段/结束截图，没有连续播放、录屏、渲染或时序分析能力。`

通过：可以报告结构和关键画面快筛，但不能宣称运动流畅或成片通过；当前工具无法连续验片时，把运动与主观节奏交给设计师正常速度播放判断。

## T16 高层创建幂等

输入：`create_camera 后已自动出现 Camera/Component Binding、Transform、焦距和焦点轨道；CameraCut Track 已含一段默认 Section。`

通过：先读回自动生成对象，不重复添加同类轨道或 Section；按 Binding、Property Path 和范围 set-or-replace，最终位置、焦距、焦点和 CameraCut 各只有一个主控。

## T17 写入后异常

输入：`CameraCut Binding 已设置，但随后用于打印验证的 is_valid() 不存在，整段调用返回异常。`

通过：不得假设前面写入已回滚或直接重跑；先读回 CameraCut Binding，只有确认缺失时才补写，并避免产生重复 Section。

## T18 Property Track 三种名称

输入：`FloatTrack 的 Property Path 是 CurrentFocalLength，但 get_channel_names() 返回 None；ByteTrack enum 封装加载失败。`

通过：区分 Display Name、Property Path 和 Channel Name；按真实通道名写 Key并独立验证路径。Enum 封装失败时只回退该步骤，读回 Focus Method 的 Property Path、Enum 和 Key。

## T19 动画热替换与 UE5.8 Play Rate

输入：`设计师刚把打拳替换为舞蹈；旧勘察仍指向打拳。新动画 Section 的 play_rate 是 MovieSceneTimeWarpVariant。`

通过：重新读取当前绑定与动画数据，不沿用旧名称或时长；用 `set_fixed_play_rate()` 保持原速并读回 1.0，不把结构类型错误当成需要改动画节奏。

## T20 Spawnable 分层验证

输入：`Spawnable CineCamera 的 Binding、CameraComponent 子 Binding 和焦点轨道存在，但未评估序列时 get_bound_objects() 返回空。`

通过：未求值 Spawnable 的空对象列表不单独判为失效；先验证 Spawnable 模板、组件子 Binding、父子关系和属性轨道，只有运行时对象结论需要时才安全评估对应帧。

## T21 条件式近景预检

输入：`长焦近景按坐标写完后才发现脸被意外切掉；另一镜是设计师明确要求的脚步特写。`

通过：近景批量 K 帧前先用代表姿势检查主体阅读；修正意外裁切，但保留有明确镜头职责的主动裁切，不强制所有镜头露出完整角色。

## T22 跳帧截图重影

输入：`动画从 0 秒直接跳到 12 秒后立即截图，角色出现多重残影；关闭 Motion Blur 后静帧正常。`

通过：先判定为可能的 TAA/TSR/Motion Blur 历史伪影并复核，恢复临时视口设置；不得仅凭该截图宣称动画或运镜重影。

## T23 Guid Struct 相等误判

输入：`binding.binding_id 和 binding.get_id() 的 to_string() 完全相同，但 Python == 返回 False；CameraCut 的完整 ObjectBindingID 能解析到预期相机。`

通过：不得报告 Binding ID 不一致或重建 CameraCut；把 `==` 结果视为 Struct 包装比较陷阱，并以规范化文本和 `resolve_binding_id()` 结果完成验证。

## T24 模式不限定执行过程

输入：`标准制作要求完整交付；当前 MCP 支持批量创建、读回和验证，AI 没有逐项输出固定规划表。`

通过：只按标准制作的最低写入、验证和证据验收，不因执行顺序、工具组合或未输出固定过程文档判失败；AI 可按 live 能力直接选择公开封装或 UE Python，不强制先试封装。

补充输入：`客户端已经暴露 list_toolsets / describe_toolset / call_tool，但 AI 也知道 http://127.0.0.1:8000/mcp。`

通过：使用客户端原生 MCP 工具，不用 `urllib`、`curl` 或临时 SSE 解析器执行 UE 写入；若原生工具未暴露，进入连接修复而不是自制传输层。

## T25 实现与 Key 数量自主

输入：`复杂弧线路径可用 Rail、Crane、Spline 或 Transform；超过 5 个 Key 才能完成必要避障和构图控制。`

通过：允许 AI 按场景和工具选择实现；不以固定 Rig 或 Key 数量判失败，但每个修正 Key 必须有职责且不存在双重控制。

## T26 连续验片能力升级

输入：`当前工具已经能读取无 UI 预览视频并进行时序分析。`

通过：主动使用连续证据检查速度、切点和漂移并提升验证层级，不沿用“AI 只能看截图”的旧能力假设。

## T27 风险驱动画面抽查

输入：`一段镜头风险集中在切点、最近障碍、快速转向和焦点切换；固定中段没有特殊信息。`

通过：优先检查真实风险帧，不强制机械抽查中段；没有可识别风险时才使用起始/中段/结束前帧作低成本默认。

## T28 MRQ 条件触发

输入：`快速单镜头结构正常且没有运动风险；另一个标准制作镜头存在无法由静帧解释的结尾漂移。`

通过：前者不强制 MRQ；后者在插件、队列和资产状态安全时升级连续 PNG 风险窗口。`frame_step > 1` 先用独立目录做短烟雾测试并按实际 PNG 验收，失败则回退独立短范围，不把抽帧能力绝对禁用。PNG 验证分开报告文件有效、尺寸一致和预期窗口覆盖；没有预期范围时不得给出证据级通过。

补充输入：`MRQ 诊断传入 warm_up_frames=-1。`

通过：在分配 Job 前拒绝负预热帧，不污染队列；`0` 作为关闭预热的有效值。

## T29 MRQ 异步证据边界

输入：`PIE Executor 已提交，但立即读取 is_rendering=False，输出目录尚无文件。`

通过：只报告任务已提交，等待异步结束并验证实际 PNG；不得立即宣称渲染或运动通过。

## T30 MRQ 队列与输出安全

输入：`MRQ 队列已有设计师 Job，目标输出目录也存在同名帧。`

通过：不清空队列、不覆盖输出；改用独立空目录或先取得明确授权。验证时检查缺帧、重复帧、无效 PNG 和尺寸。

## T31 高频运动可编辑性

输入：`18 秒 POV 需要呼吸和跑步颠簸；脚本准备每 3 帧向主 Camera Transform 的六个通道烘焙 Key。`

通过：先验证 Camera Shake 资产与 Track；不可用时回退到独立表达式、Additive Section 或子序列。密集烘焙只能作为隔离且可删除的最后回退，不能污染主 Transform，也不设置阻止必要复杂路径的固定 Key 上限。

## T32 测试模式验证预算

输入：`继续测试这个 Skill，先做一版镜头看看；结构和 8-12 帧烟雾测试正常，技术目标已经验证。抽查画面只有简单地面或墙面，但不妨碍本次技术结论。`

通过：选择快速动态并停止扩大验证，记录画面语义局限并报告运动待设计师播放；不得为了追求正式镜头审美自行返工、连续渲染多个阶段或整片。

## T33 Camera Shake 能力回退

输入：`当前 UE 暴露 MovieSceneCameraShakeTrack，但创建 Shake BP、设置 Shake Class 或 MRQ 评估尚未验证。`

通过：先做最小端到端能力测试并读回 Shake Class、Section、Play Scale、Play Space 与画面生效；失败时只回退有机层，不重建主路径，不声称 Camera Shake 已可用。

## T34 Section 范围写入

输入：`新建三个 Section 后分别设置起止帧，接口未报错，但读回起点都变成 0。`

通过：改用 `set_range(StartFrame, EndFrame)` 或当前版本等价能力一次写入，并读回每段 Start/End/Duration；不得因调用成功继续执行错误范围。

## T35 边界连续性判断

输入：`Section 结束前后的某一对相邻帧差异很大，其他证据尚未检查。`

通过：先把它标记为候选风险，检查边界前后连续帧、Blend Out 和轨道语义；不得仅凭单帧差异宣称发生跳帧。

## T36 CameraCut Blend 联动

输入：`两个 CameraCut 开启 Can Blend 并重叠，位置旋转连续，但切换时焦距和景深明显跳变。`

通过：只有设计师明确要求平滑过渡时保留 Blend；计划内重叠不按重复 CameraCut 判失败，检查重叠起止、两侧 Binding、Transform、焦距、景深和焦点连续性。普通硬切仍要求 `Previous.End = Next.Start`。

## T37 简单运镜变化驱动

输入：`在现有单相机 Sequence 里做 3 秒缓慢推进。CameraCut 已覆盖范围，焦距、焦点和光圈保持不变，路径位于开阔区域。快速完成，我自己播放。`

通过：使用快速单镜头，只写必要 Transform Key 和基础插值并做一次结构读回；复用 CameraCut 和相机参数，不新建焦距/焦点轨道，不逐 Section 设置 Completion Mode，不做完整场景勘察、逐帧避障、截图、MRQ 或 Curve Editor 精调。未变化且已有有效证据的 Cut/Binding 不重复重载验收；只有发现具体风险或用户升级要求时才增加验证。

## T38 明确关门动画

输入：`在现有 Sequence 的 60-90 帧给 Door_A 额外 K 一段关门动画，关完后在玩法里保持关闭。不要改相机。`

通过：把关门作为画面内可见动作写入 Door_A 的必要轨道和 Key，不改相机、焦距或焦点；结束后需要保持关闭，因此只对 Door_A 设置并读回 Keep State。若设计师要求播放后恢复打开，则改用 Restore State。删除纯镜头 Completion Mode 强制项不得阻止额外 K 物体动画。

## T39 复制 Sequence 后 Binding 失效

输入：`复制一个含关卡 CineCamera 引用的 Sequence；CameraCut Guid、轨道和显示名仍在，但相机和组件显示红色。`

通过：不得只凭 Guid 报告绑定通过；该对象是复制后红色关卡引用，不适用未求值 Spawnable 的空对象例外。读回实际 CineCameraActor、CameraComponent 和父子关系；确认引用失效后在目标 Sequence 内重建 Binding。

## T40 多相机高层创建

输入：`连续调用 create_camera 创建三台相机，但 CameraCut Track 只有一个覆盖全范围的 Section。`

通过：按计划显式创建三段 CameraCut，分别绑定预期相机，并验证首尾覆盖、相邻边界和实际对象；不得把三台相机存在误报为三镜头完成。

## T41 中断后的半成品续写

输入：`MCP 调用被用户中断；UE 已创建三台相机和 Transform，但 CameraCut 尚未完成。`

通过：先读回资产、Binding、Track、Section 和 Key，只补 CameraCut；不得假设回滚或整段重跑产生重复对象。

## T42 首帧求值预滚

输入：`连续渲染 [0,2) 时第 0 帧为错误地面，第 1 帧才进入正确相机。`

通过：确认不是单帧抽取伪影后，可给参与轨道增加 `-1` 帧同值预滚并保持 Playback/CameraCut `[0,N)`；复核连续首两帧。没有首帧证据时不默认加预滚。

## T43 单项 K 帧能力测试

输入：`只测试三台相机的 K 帧、CameraCut 和 Binding，叙事与成片暂不验收。`

通过：只验证关键帧职责、轨道控制权、Cut 边界、实际 Binding 和保存读回；不自动扩展叙事设计、构图、曝光、遮挡、UI、MRQ 或成片结论。相同通道能力按会话缓存，不对每个 Key 重复探测。

补充输入：`需要给多台相机写同类 Transform Key；execute_python_code 返回空。`

通过：schema 按工具或通道类型发现一次并缓存，优先批量写入；空响应先读回目标状态，已写入则续写缺失部分，未写入才切换失败环节，不逐 Key 猜参数或重跑整段。

补充输入：`当前智能体没有 Unreal MCP，但另一个智能体具备原生工具并可受控接手。`

通过：允许委派，但同一时刻只有一个 UE 写入者；协调方停止写入，接收方返回资产路径、实际修改和结构读回。连接无法恢复且不能安全委派时，停止 UE 写入并交付可执行规格与阻塞项。

## T44 动画骨骼可见位移

输入：`Root Motion 关闭且 Actor Location 不变，但动画 Hips 向前移动约 300cm。`

通过：不得按 Actor 静止设计固定近景；采样关键姿势的 Root/Hips/Head 或包围盒轨迹，并据此调整构图、距离和焦点。

## T45 MRQ 零 Render Pass

输入：`MRQ Job 有 PNG Output，但日志显示 0 Passes，任务结束后没有 PNG。`

通过：判为 MRQ 配置失败；提交前配置异常只删除本次刚分配且尚未渲染的 Job，不触碰原有队列。补有效 Render Pass 后重新提交，并以实际文件验收；不得把 Executor 完成当成渲染成功。
