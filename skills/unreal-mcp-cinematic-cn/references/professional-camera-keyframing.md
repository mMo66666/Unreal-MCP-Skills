# 专业摄像机打帧规范

用于把镜头语言转成 UE5 Sequencer 可执行、可验证、可返工的摄像机关键帧方案。

## 核心原则

- 专业运镜不能只留下没有职责的起止 Transform。根据设计师已给出的镜头意图，判断是否需要起势、慢入、主运动、语义变化、减速或落稳；这些是候选阶段，不是每个镜头必须凑齐的清单。设计师没有给出语义变化、停顿或落稳意图时，不得为了显得完整而自行补充。
- Camera Path、朝向和焦点必须分开写，并遵守 `ue5-camera-implementation.md` 的“相机控制权唯一”；不要让 Transform、Rig、Look At、Rotation 或两套焦点系统抢控制。
- 普通运镜的 Key 密度应与空间路径、节奏和控制方式匹配；优先用主控 Key、关键帧间距和曲线插值解决节奏，不默认使用 Time Dilation 或 Play Rate。
- 如果工具暂时不能精确设置曲线，只添加有明确目的的修正 Key，不要用大量中间 Key 把曲线切碎。
- Dolly Zoom 不要同时启用 Look At Tracking 和密集 Rotation Key；二选一，避免双重控制导致抖动。同步处理 ManualFocusDistance，避免主体发虚。



## 运动分层：主运动与高频有机层

- 主运动层：位移、朝向、焦距、构图的大形态，继续使用少量主控 Key、曲线插值、Crane/Rail/Spline 路径控制。
- 高频有机层：呼吸、步伐、手持感、载具震动、受击震动等周期性小幅运动，优先使用独立 Camera Shake 资产与 Sequencer Camera Shake Track；写入前先做能力检查，写入后读回 Shake Class、Section Range、Play Scale、Play Space 和 Binding。
- Camera Shake 的 Root Pattern 选择按 `ue5-camera-implementation.md` 执行；不同频率或运动语义通常使用不同 Shake 资产/Section，不能只靠 Play Scale 冒充频率变化。
- Camera Shake 不可用时，依次考虑独立表达式、Additive Transform Section 或隔离的子序列。脚本密集烘焙是最后回退，只能放在可单独关闭、删除和重建的有机层，不得写进主 Camera Transform。
- 读回时检查轨道职责和 Key 分布；多个 Transform 通道出现长区间周期性密集 Key，视为可编辑性风险。它不设置固定 Key 数量上限，但在未隔离、未说明和未验证返工路径时，不得报告为标准制作完成。
- 使用高频有机层时，必须在执行规格或报告里写明实现手段、频率、振幅、作用轴和衰减方式；如果只是粗测，可写“临时 Shake 参数，待设计师验片校准”。

POV / 手持默认参考值，可按项目校准：平静呼吸约 0.2-0.3Hz，紧张呼吸约 0.5-0.8Hz；走路步频约 1.5-2Hz，跑步约 2.5-3Hz，垂直 bob 为主、左右 sway 为辅；贴地视角约 15-25cm，站立眼高约 160-170cm；“呼吸变重”应表现为频率和振幅上升，并持续一小段时间再衰减。

## 转头 / Pan 自然包络

- 设计师要求自然转头、环顾或快速 Pan 时，Rotation 曲线可用“加速 - 短匀速 - 减速停”；设计师明确要求机械匀速或其他质感时按原意执行。
- 大角度转头控制峰值角速度，参考 200-300°/s；按项目风格可调整，但不能瞬间甩完导致信息不可读。
- 到位后允许轻微过冲回稳；连续环顾时，每段转头之间保留约 0.2-0.5s 的短暂停顿。
- 转头自然度优先通过 Rotation 曲线、Look At 目标和少量有职责 Key 解决，不要用无目的 Location Key 修转头。
## Rotation Yaw 连续化

Orbit、Crane、Rail 环绕镜头在烘焙 Rotation Key 时，必须保持 Yaw 数值连续，避免跨过 `180 / -180` 断点产生反向抽动。

正确示例：`-145 -> -180 -> -215`，或 `215 -> 180 -> 145`。错误示例：`-145 -> 180 -> -145`。写入 Sequencer 前检查 Yaw unwrap；如果 Curve Editor 出现垂直跳变或大幅反向曲线，优先检查 Yaw 是否没有连续化。

## 环绕镜头 Rotation Key

Crane / Rail 与 CineCamera Rotation 的主控关系、中段 Rotation Key 准入和控制权切换以 `ue5-camera-implementation.md` 为唯一宿主。本文件只负责 Yaw 连续化、Key 职责和曲线质量，不维护第二套准入条件。

## 数据判断边界

先检查当前工具能否读取 Curve Editor、连续播放、录屏、渲染或等价时序证据；不能时，Key 数量、Section Range 和截图只能帮助定位问题，不能单独证明运镜流畅。

- 结构验证通过不等于运动流畅。
- 截图能判断构图、遮挡、穿模和主体可读性，但不能证明卡顿、顿挫、速度连续性。
- 运动流畅性必须通过连续播放、录屏、渲染或等价时序证据验证；当前能力支持时由 AI 主动分析，只有数据和抽帧时只能说“结构和关键帧规则通过”，并由设计师提供正常速度播放反馈。
- 如果用户反馈卡顿、顿挫或不顺，第一反应是检查并删除无目的中间 Key、检查 Look At / Rotation 曲线，而不是继续加 Key。

## Camera Path 控制规则

Rig 选择、位置/朝向/焦点主控和控制权切换以 `ue5-camera-implementation.md` 为唯一宿主。本文件只要求 Key 与所选路径职责一致：主控表达路径和节奏，修正 Key 只解决已确认的避障、构图、焦点或节奏问题；不设固定数量，也不同时用世界 Transform 与 Rig 参数争夺同一位置路径。

## 简单运镜原语

缓慢推进、快速拉远、单轴横移、升降、简单摇摄/甩镜等路径和目标明确的单镜头，优先使用最小主控集合：通常为起止 Key 加基础插值；只有节奏转折、落稳、避障或构图重锁真实存在时才增加 Key。焦距、焦点、光圈和景深未变化时继承现状，不为显示完成度创建轨道。快速单镜头默认只读回参与的 Transform/Rig、范围和 Binding，不自动进入 Curve Editor、截图或连续渲染。

## 运动规划表

复杂镜头需要拆解、返工诊断或交接时可使用运动规划表；它不是每次执行必须输出的文档，也不是逐项打 Key 或自动补阶段的模板。只有单镜头确实包含多个已给阶段时，才分别写起止时间、运动类型、包络和过渡方式。

| 帧号 | 阶段 | Camera Path | Look At Path | 焦距/焦点/景深 | 速度意图 | 画面检查 |
|---|---|---|---|---|---|---|
| 起始帧 | 起势 | 起点构图 | 初始目标 | 建议值 | 停稳/慢入 | 空间和主体可读 |
| 10%-20% | 慢入 | 离开起点 | Look Ahead | 建议值 | 轻启动 | 不要突然冲出 |
| 35%-70% | 主运动 | 主要路径 | 语义目标 | 建议值 | 加速/掠过 | 前景短暂，不挡主体 |
| 70%-90% | 减速 | 接近终点 | 收束目标 | 建议值 | 回收速度 | 避免贴面/穿模 |
| 结束前最后有效帧 `EndFrame - 1` | 落稳 | 终点构图 | 稳定主体 | 建议值 | 停顿 | 结尾可读 |

上表只示范“设计师已经要求起势、慢入、主运动、减速和落稳”的镜头。匀速、机械、无停顿或无落稳镜头只填写实际存在的阶段，不得照表补齐。

## 默认禁止保险 Key

快速单镜头中，除非设计师明确要求，或结构/画面验证证明失败，否则普通 Dolly、Push、Pull、Dolly Zoom 不默认添加过渡 Key、中段辅助 Key、光圈 Key 或多余 Rotation Key。

低风险普通运动可从 Location、Focal Length 和 ManualFocusDistance 的起止主控开始；Rotation 与额外 Key 按主体阅读、路径和构图需要添加。每个新增 Key 都应能说明它解决的避障、重锁构图、焦点、前景掠过、节奏或落稳问题。
## 主控 Key 与修正 Key

- 流畅优先来自少量主控 Key 和与设计师运动质感一致的连续曲线，不是 Key 越多越专业；S 曲线只是可选实现，不是默认目标。
- 普通推镜、拉镜、平移、升降从最小有效主控集合开始，按落稳、构图和路径需要增加 Key。
- 改变方向、绕障碍、低空穿行、环绕或前景掠过时，自主选择 CameraRig_Crane、CameraRig_Rail、spline path 或 Transform；Key 数量以完整表达路径且不破坏连续性为准。
- 修正 Key 只在有明确目的时添加，例如避开遮挡、重新锁定构图、控制前景掠过、制造节奏停顿或进入最终构图。
- 每个修正 Key 必须能说明“解决什么问题”；没有明确功能的中间 Key 会破坏曲线连续性。
- 只有设计师规格包含起势或落稳时，才在开头保留相应起势、在结尾提前进入最终构图并轻微落稳；未要求时不自动补这两个阶段。

## 中间 Key 判断规则

中间 Key 不是绝对禁止，但添加前必须先判断它解决的是速度曲线问题，还是空间路径问题。

- 速度曲线问题：设计师已要求的慢入、快出、缓出、停顿或落稳没有被正确表达，或执行无意中套用了与规格不符的匀速/默认 S 曲线。优先处理插值、切线、曲线类型或不等距 Key 时间布局，不优先增加 Transform 中间 Key；设计师明确要求匀速时不视为问题。
- 空间路径问题：避障、前景短掠过、路径转向、构图重锁、主体防出画、提前进入最终构图或结尾落稳。可以添加中间 Key，但必须说明画面职责。
- 如果一个中间 Transform Key 只是为了改变速度，却同时改变了摄像机空间路径，通常是错误插帧，优先删除或改用插值 / 切线 / Rig 参数。
- 如果说不清这个 Key 删除后会造成什么画面问题，它就是保险 Key，应删除。

## 速度和前景

- 当设计师已经要求完整的起势、主运动、减速和落稳，但没有给出具体比例时，可把 `0%-10% / 10%-35% / 35%-70% / 70%-90% / 90%-100%` 作为执行假设并明确标注；不得把这组比例套给匀速、机械、突然切入或没有落稳要求的镜头。
- 前景掠过只允许短暂服务层次和速度；持续时间按 Display Rate、镜头长度和主体阅读判断，不套固定帧数。
- 前景遮挡的经验风险与失败判断统一按 `cg-staging-lighting.md`；本文件只检查相关 Key 是否确有前景控制或主体可读职责。
- 画面只剩墙面、屋顶、树冠、地面或天空，判定为遮挡失败。

## 曲线类型不要模板化

不要每个镜头都使用同一种对称 S 曲线。曲线要服务镜头功能：

- 对称 S：缓入缓出接近对称，适合中性移动、过渡镜头、平静观察。
- 慢入快出：前段蓄势、后段推进，适合揭示、情绪推进、危险逼近。
- 快入慢出：起步较快、长缓出落稳，适合结果揭示、拉开、升高和结尾收束。
- 恒速加缓出：中段相对稳定、尾部减速，适合穿行、掠过、跟随视线确认。
- 短缓入或接近线性：可用于设计师明确要求的冲击、动作切入、突然反应、机械匀速或持续观察；它不是默认曲线，也不得因镜头类型标签擅自采用。

多镜头组合里，曲线节奏要有差异；如果每段曲线形状几乎一样，判定为模板化运镜，需要按镜头功能重设速度意图。

曲线差异必须来自镜头功能和情绪，不是为了变化而变化。如果连续两个镜头承担相近功能或情绪，可以使用相近曲线；只有整段镜头无判断地套同一种曲线时，才判定为模板化问题。

UE5.8 暴露了 Sequencer Key 插值和切线相关 API 时，优先用插值 / 切线控制曲线，而不是为了节奏硬加空间中间 Key。可尝试的方法包括 `set_interpolation_mode`、`set_tangent_mode`、`set_arrive_tangent`、`set_leave_tangent`、`set_arrive_tangent_weight`、`set_leave_tangent_weight`；常见模式包括 `SMART_AUTO`、`AUTO`、`USER`、`BREAK`、`LINEAR`、`CONSTANT`。写入后必须读回验证该通道是否真的接受设置。

如果某个 Transform、Float、FocalLength 或 ManualFocusDistance 通道不支持这些方法，或写入后读回不一致，不要声称已精确设置 Curve Editor；退回不等距 Key 时间布局表达曲线原型。例如落稳型镜头可让主要位移在 70%-85% 前完成，再用结尾短段落减速落稳；只有第 3 个 Key 有明确节奏或构图功能时才添加。


## 空间路径流畅性

- 短镜头若没有静止开场要求，避免因误写 Key 产生无意的过长停帧；这不代表必须自动加入短起势。
- 运动镜头优先按“空间路径”设计关键帧，而不是只分别调整 X/Y/Z 曲线。X/Y/Z 单独 Auto 出不同弯曲时，画面容易拧、飘、顿。
- 普通运动镜头优先用少量主控 Key 形成与设计师意图一致的连续曲线；只有需要避障、改方向、重锁构图或设计师已给出的明确节奏点时才添加修正 Key。
- Camera Path 和 Look At Path 分开：摄像机走空间路径，视线看目标点或空对象，并允许轻微 Look Drift；不要把旋转和位置都机械锁死在同一条曲线上。
- 设计师要求结尾落稳时，先进入最终构图再做轻微落稳，不要把大位移压到最后几帧完成；没有落稳要求时，只检查结尾运动是否符合原规格。
- 验证时看合成后的空间速度和画面运动，不只看每条 Location 曲线是否 Auto/Bezier。

## 运动验证

执行后除了 CameraCut，还要检查：

- Transform Key 是否都有明确功能；是否为了“更专业”无目的地堆中间 Key。
- 主控曲线是否连续并符合设计师指定质感，是否被无目的中间 Key 切碎；全程匀速直线只有在违背已确认规格或出现非预期漂移/顿挫时才判为问题。
- 设计师规格要求慢入、停顿、减速或落稳时，这些节奏是否被准确执行；未要求时不得把缺少这些阶段判为失败。
- Camera Path 和 Look At Path 是否职责清楚且与规格一致；未要求 Look Ahead 或 Look Drift 时，不得把缺少它们判为失败。
- 每镜的位置、朝向和焦点是否各只有一个主控；控制权切换是否在指定帧关闭旧主控并保持连续。
- 设计师规格要求中段新增信息或切点构图变化时，执行是否实现；未要求时不自动增加。
- 设计师规格要求前景层次时，前景是否成立且没有长期遮挡；明确要求无前景时不属于失败。
