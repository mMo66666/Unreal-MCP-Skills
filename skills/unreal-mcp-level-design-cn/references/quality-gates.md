# 地编质量 Gate

在阶段验收、评审或最终交付时读取。根据任务类型裁剪，但不要用一个维度替代另一个维度。

## A. 结构正确性

- World、关卡、Data Layer、资产包路径正确。
- Actor 类、数量、标签、Transform、组件和引用符合计划。
- 资产真实 Class 已读回，没有根据命名前缀选择错误的生成方式。
- 陌生或大批资产集已在隔离 Asset Zoo 中按类别和尺度锚点走查，正式场景未批量使用未验证资产。
- 样条连通，Landscape/Water/PCG/Foliage 输入有效。
- 资产已按需保存、编译，无未解释的 dirty 状态。
- 创建过程幂等，再次运行不会无止境复制对象。

## B. 尺度与空间

- 玩家胶囊、镜头、门、通道、楼梯、掩体、车辆和模块比例一致。
- 小型道具、家具、建筑构件和载具之间无语义尺度倒挂；厨具大于座椅、门把手接近门高、路灯矮于人物等候选已用尺度锚点和局部视图复核。
- 模块套件已通过回环、堆叠和缝隙压力测试，内外角、门窗、多层、斜接与碰撞组合可用。
- 可玩边界、高差、坡度和落差符合移动能力。
- 建筑和道具底面由世界 Bounds 对齐到有效地表；不存在统一 Z 值造成的漂浮或埋入。
- 整栋建筑间无非设计性 Bounds 重叠；有意模块接缝已有收口，且无可见穿模、Z-fighting 或碰撞阻断；其他候选已用局部视图复核。AABB 粗筛不替代形状判断。
- 路径长度换算出的行进时间符合节奏目标。
- 没有非设计意图的死路、软锁、不可回返区或越界捷径。

## C. 动线与玩法

- 主路线清晰，备选路线和回环数量符合体验目标。
- POI、出生、目标、瓶颈、制高点和安全区关系可解释。
- 空间可供性足以传达基础移动、交互和危险，不依赖文字或小地图才能理解。
- 关键视线有遮挡、揭示和风险回报；不存在偶然的全图狙击线。
- 战斗空间有入场读场时间，并检查掩体、侧翼、交叉火力和撤退；探索空间检查地标、诱导和发现节奏。
- 难度问题已优先从站位、路线、高差、视线和资源位置修正，而非直接堆叠敌人数值。

## D. 视觉与叙事

- 玩家视角、俯视和远景轮廓都能读出空间层次。
- 道路、地块、建筑和功能道具形成空间闭环；主要建筑有可解释的临街面、入口、后场与边界。
- 道路端点均有连接、回车、延伸或边界收口，不裸停在临时承载面。
- 地标不被植被、雾、曝光或背景噪声吞没。
- 相关道具形成可解释的功能/叙事簇并保留负空间；战斗区域的人物高度可读带未被非必要杂物遮挡。
- 密度存在焦点、过渡和留白，无全场机械等距、无意孤立资产或与目标成熟度不符的大面积裸露测试地面。
- 相邻资产的纹理尺度协调，无明显 UV 拉伸、接缝或平铺周期；近、中、远距离均已复核。
- 生态、建筑、道路和故事痕迹共享同一世界逻辑。
- 至少查看全场高位俯视、45° 鸟瞰和玩家视角；Lit 截图有时序残影时已用 Unlit 复核几何并恢复原视图。
- 已查看截图本身；“截图工具成功”不等于视觉通过。

## E. 技术与性能

- PIE 可进入，关键路径无阻断碰撞、穿模、导航或触发问题。
- PIE 前已验证 PlayerStart 和关键路线下方存在可碰撞承载表面，并以预期 Pawn/胶囊底部复核净空、出生占位和阻挡；不存在悬空、埋地或初始重叠，否则明确标为未验证。
- 日志无未处理的 error、compiler error、missing asset 和部分失败。
- World Partition / Data Layer 在预期区域正确加载。
- 高密度对象优先实例化或程序化管理，避免大量无意义独立 Actor。
- 性能先判断 CPU/GPU 瓶颈，再针对性优化；记录测量环境和数据。

## F. 安全与可回滚

- 修改对象范围与用户授权一致。
- 未删除、覆盖、批量移动或重命名未授权正式资产。
- 阶段检查点、Data Layer、标签或变更清单足以定位本轮修改。
- 超时/失败后先读回，没有重复执行造成二次生成。

## 最低证据包

最终汇报至少包含：

1. 目标 World/资产路径。
2. 创建与修改对象的摘要数量和关键名称。
3. 一项结构读回证据。
4. 至少一组基线/完成截图或对应视角说明。
5. 涉及可玩性时的 PIE 结果。
6. 进入正式美术包装时的白盒试玩记录：关键路径完成情况、计划与实测时长偏差、实际采用路线。
7. 涉及开放世界、PCG 或大量植被时的流送/性能结果。
8. 未验证项与风险，不得省略。

## 失败处理

- 结构失败：回到产生该对象的阶段修正并重新读回。
- 视觉失败：明确是构图、尺度、密度、材质、灯光还是遮挡问题，只改相关变量后重新截图。
- PIE 失败：停止继续装饰，先修阻断性碰撞、导航、加载或脚本错误。
- 性能失败：先依据 CPU/GPU 判定选择方向，不进行无目标的“全面优化”。
- MCP/工具失败：保留当前关卡状态，按最小连接诊断处理，不切换到临时 HTTP 资产写入。
