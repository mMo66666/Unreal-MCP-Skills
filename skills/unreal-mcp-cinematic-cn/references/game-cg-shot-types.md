# 游戏 CG 类型识别

先识别设计师给的视频属于哪类，再整理执行镜头。不要把所有需求都套成普通飞行展示。

## 常用类型

- 地图/关卡展示：按设计师给的空间、路径、区域细节、核心区域和收束画面执行。
- 角色登场：按设计师给的环境预兆、局部细节、身份揭示、标志动作和情绪定格执行。
- Boss 登场：按设计师给的异常预兆、尺度暗示、威胁动作、全貌揭示和玩家目标回收执行。
- 技能/大招展示：按设计师给的施法者、蓄力、释放、命中/影响范围、余波和回到可读战斗状态执行。
- 武器/载具展示：按设计师给的轮廓、材质/机构细节、功能动作、威力或速度展示执行。
- 剧情过场：按设计师给的目标、阻碍、冲突升级、转折、结果和反应执行。
- Trailer/宣传片：按设计师给的世界观、卖点、节奏升级、视觉高潮和收束画面执行。
- 买量/投放视频：按设计师提供的钩子、爽点、故事冲突和记忆画面组织镜头。
- 实机演示向：信息可读优先，镜头服务玩法路径、操作反馈和玩家方向感。

## 选择规则

- 用户没说类型时，可以根据设计师文本、主体和用途做暂定分类并明确标注；仍不确定时写“类型待确认”。只有类型会影响执行规格时才询问，不默认改成场景展示或技术验证视频。
- 类型识别只服务执行，不替设计师改创意。
- 如果同一需求包含多类目标，先确定主类型，再把其他类型作为执行约束。
- 买量视频不替设计师发明爽点；缺少钩子、爽点或记忆画面时，先问清楚再做。
