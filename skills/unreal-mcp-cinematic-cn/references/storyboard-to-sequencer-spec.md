# 分镜到 Sequencer 规格

把设计师分镜、镜头表、分段画面描述或脚本转成下游执行 Skill 可以直接使用的规格表。不要只写“电影感”“慢推”“震撼”，要给出可执行字段。

## 输入原则

- 优先保留设计师原文意图，不擅自改剧情、爽点、角色关系或最终记忆画面。
- 缺少执行信息时，只补技术字段：景别、机位、焦距、Look At、CameraCut、关键帧、绑定 Actor、经设计师授权的占位方案和验证点。
- 只有创意归属不明确，且不同选择会改变剧情、镜头数量或核心意图时，才询问是否允许 AI 补分镜。设计师明确说“发挥”“直接做”或同义授权时，自主补齐镜头与技术规格并报告关键假设。

## 规格总览

每个镜头至少包含：

- 执行模式：静态分镜、快速单镜头、快速动态、标准制作或成片检查
- 画幅/输出比例：按 `ue5-camera-implementation.md` 的“画幅与 Filmback”执行
- 设计师原始描述
- 镜号
- 时间/帧范围：执行规格统一使用 `[StartFrame, EndFrame)`，并写明持续帧数
- 镜头目的或画面重点
- 景别
- 摄像机类型
- Camera Path 控制方式：Transform、CameraRig_Crane、CameraRig_Rail 或 spline path
- 焦距与焦点建议
- 机位和运动
- Transform Key 策略与 Crane / Rail / Spline 策略；POV、手持、呼吸、步伐或震动要标注高频有机运动实现方式
- Look At 目标和焦点目标
- 角色朝向要求：按 `ue5-camera-implementation.md` 的“角色朝向”执行
- 绑定 Actor 或筛选规则
- 角色动画/对时间：动画资产或经授权的占位动作、关键动作帧、CameraCut 对齐点
- CameraCut 区间
- 需要 K 的属性：Camera Transform、焦距、手动焦点距离、景深、Actor Transform、可见性等
- 是否修改场景 Actor
- 结束后状态（仅状态相关对象）：恢复 / 保持 / 项目明确默认值
- 画面安全：曝光、遮挡、编辑器 UI、主体占比、穿模风险
- 验证点

简单运镜或单技术点可以把未参与的焦距、焦点、光圈、景深、CameraCut 和构图关系标记为“继承当前有效状态”；“至少包含”要求说明控制归属，不要求为未变化维度新建轨道、Key 或专项验证。

## 运动规划表

复杂镜头需要拆解、返工诊断或下游交接时，可使用 `professional-camera-keyframing.md` 的运动规划表；简单镜头或执行器能够直接表达并验证时不强制输出过程文档。

## 时长分配

设计师给出的总时长、逐镜时长、帧段、动作关键帧和固定切点优先级最高，除非设计师明确授权调整，否则按原值执行。

只给总时长但未给逐镜时长时，AI 根据动作关键姿势、信息量、情绪变化、镜头职责和切点连续性自主选择均分或非均分，并报告重要执行假设；不套固定镜头类型时长表。设计师授权“你来发挥”时，AI 可以自主决定总时长或逐镜时长，但仍尊重动画实际帧率、资产长度和交付规格。

总时长、动作长度和交付要求均无法确定，且不同选择会显著改变结果时，才询问设计师。

## 执行字段规则

- 镜号用 `SH010`、`SH020`、`SH030`，保留插入空间。
- 概念方案可用秒；执行规格必须换算成帧或说明按项目帧率换算。
- 帧率默认使用项目 Sequencer 默认帧率；设计师明确要求时才覆盖。
- 帧区间、持续帧数、最后有效帧、Display Rate、Tick Resolution 和动画采样率统一按 `ue5-sequencer-basics.md` 的“时间区间契约”解释。禁止用 `0-104` 这类写法掩盖结束帧是否包含。
- 画幅/输出比例以 `ue5-camera-implementation.md` 的“画幅与 Filmback”为宿主；先读上下文和项目设置，交付比例仍不明确时再问。
- 角色正脸轴以 `ue5-camera-implementation.md` 的“角色朝向”为宿主；先使用设计师信息或候选视角证据，仍不明确时再问。
- 电影/过场用 `CineCameraActor`；用户明确不要摄像机时，标注“无摄像机，仅 Actor 动画”。
- 多镜头需要 CameraCutTrack。
- 只列真正参与镜头或动画的 Actor，不默认绑定地面、隐藏物或无关 Actor。
- 角色动作、转身、攻击、命中、落地等影响切镜时，按 `ue5-character-animation.md` 补动画资产、经授权的占位动作、关键动作帧和 CameraCut 对齐点。
- 动画时长不匹配按 `ue5-character-animation.md` 处理，本文件不维护第二套裁切、变速或留白规则。
- 默认只写 Sequencer，不直接改变场景资产的持久摆放；运行时状态由 Completion Mode 决定。
- 如果需要生成、删除、移动 Actor，必须单独标注并等待确认。
- 状态恢复按 `ue5-sequencer-basics.md` 的 Completion Mode 执行，不得默认在最后有效帧 K 回原 Transform。画面内回位只有设计师明确要求时才作为动作写入。

## 交付格式

给执行器的摘要复用“规格总览”字段，并只补资产级信息：LevelSequence 路径和文件名、需要创建或复用的相机/Actor、CameraCut 覆盖、允许的场景修改、必要结束状态与本次验证深度。复杂镜头、角色动画或高频有机层再追加对应 reference 要求；简单镜头不重复输出未参与字段。

如果用户已经表达 UE 落地、Sequencer、MCP 执行、直接创建或继续制作等意图，不要停在方案。执行模式按 `mcp-cinematic-execution.md` 的意图映射自动选择；先做最小目标定位和能力检查，只有主体、路径或风险不明确时才升级场景勘察，只有角色方向、交付规格或不可逆资产操作仍不明确时才提问。

如果用户已经有场景或上一轮已经生成占位场景，执行规格必须标注“复用现有场景，不重新搭建”。除非用户明确要求补场景或重搭，只创建摄像机、LevelSequence、CameraCut 和必要轨道。
